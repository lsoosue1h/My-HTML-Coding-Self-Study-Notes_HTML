Notes 3

목차 list

li 태그는 목차(list)를 만든다. ul 태그(unordered list)는 li를 그룹핑하는 부모태그이다. 즉, 목차들을 서로 구분한다. 또 다른 예시

  • milk
  • eggs
  • phone case
  • pen
li 태그는 반드시 부모 태그를 가지고, ul 태그는 반드시 자식 태그를 가진다.

불릿(•)으로 표시하는 ul태그와 달리, ol 태그(ordered list)는 순서(1,2,3)가 있는 목록을 만든다. 넘버링은 자동으로 된다. 즉, 목록 중 하나를 삭제해도 숫자는 자동으로 업데이트 되어 일일이 수정할 필요가 없다.

문서의 구조.

title 태그로 웹페이지의 제목(책으로 비유하면 표지)을 설정한다. 검색 엔진들이 이 태그를 제목과 같은 정보로 이용한다. title태그를 쓰지 않으면 노출도, 접근성에서 큰 손해이다.

UTF-8은 인코딩의 한 종류이다. 인코딩 encoding은 컴퓨터가 정보(자연어 등)를 약속된 규칙에 의해 변환(0과 1로)하는 과정을 지칭한다.
메타 meta 태그는 웹 페이지의 제목, 문자 인코딩, 페이지 설명, 작성자 등과 같은 정보를 제공하는 역할을 한다. charset의 cara는 문자(character), set는 집합이라는 뜻이다. 다시 말하자면 브라우저에게 utf-8로 읽으라고 하는 것이다.

title, meta 같이 본문을 설명하는 태그들은 head 태그로, 본문은 body 태그로 묶는다. 고위직 태그(head, body)를 감싸는 최고위층 태그: html 태그.

Notes 4

웹 서버 사용하기; 두 가지의 업로드 방식

1. 웹 호스팅(깃허브 페이지로)

사실 깃허브는 개발자들이 소스코드를 백업하고, 그것을 매개로 서로 협업하기 위한 사이트다. 웹 호스팅 서비스도 제공하기에 유용하다. 지금 보고 있는 이 사이트도 깃허브로 호스팅 되었다.

깃허브 웹 호스팅을 사용하려면 우선, 계정을 만들어 가입하고 로그인을 한다. 그 다음, 저장소(repository)를 만들어야 한다. 왼쪽에 [new] 버튼(저장소가 없을 시 create repository)이나 우측 상단의 (+)버튼, [new repository]순으로 클릭한다. 마지막으로, [create repository] 버튼을 클릭해 저장소를 생성하면 된다.

이제 저장소에 소스코드를 업로드하면 된다. 'Quick setup'(파란 부분)의 하단을 보면 두 개의 링크: creating a new file과 uploading an existing file이 있다. 이미 파일이 존재하므로 후자를 클릭해야한다. 업로드할 소스코드를 택하고 열기를 클릭한다. 업로드가 끝나면 'Commit Changes' 아래에 설명을 입력하고 초록색 Commit Changes 버튼을 클릭해 저장한다.

추가: 깃허브는 index.html.을 기본 홈페이지(홈 화면/ 메인 화면 역할)로 인식한다. 그러므로 index.html은 필수적이다!

이제 웹사이트를 운영해 보자. 깃허브 저장소에 들어가 상단의 설정 탭을 클릭한다. 저장소를 설정할 수 있는 화면이 나올 것이다. 왼쪽 사이드바에 있는 pages 메뉴를 클릭하면 웹 호스팅을 할 수 있는 GitHub Pages가 나온다. 여기의 [Source] 항목에서 'deploy from a branch' 선택하고 바로 아래에 있는 [branch]는 main, [Folder]는 root를 선택한다. [Save] 버튼을 누르면 웹 서버 기능이 활성화 된다.

설정 탭 옆에 Action 페이지로 진행과정을 볼 수 있다.

파일 변경하고 그것을 반영하는 방법. 저장소를 클릭해 저장소 페이지로 간다. 거기에서[add file], [upload file] 수능로 클릭하고 다시 파일들을 드래그 앤 드롭하고 저장한다.
